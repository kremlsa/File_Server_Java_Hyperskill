Description

An HTTP File Server is like remote file storage. You can access your files from anywhere in the world if you have Internet access. In this project, you will write your own file keeper that will let you save, access, and delete files.

Let's start by simulating a file storage that can save only 10 files named file1, file2, file3, ... file10. At first, when the file server just starts, there are no files: you add them using a command add NAME, where NAME is the actual name of the file. If this file already exists or has the wrong name, output Cannot add the file NAME; otherwise, output The file NAME added successfully.

If you want to retrieve the saved file, input get NAME, where NAME is the actual name of the file. If this file exists, output The file NAME was sent; otherwise, output The file NAME not found.

If you want to delete the file, input delete NAME. If this file exists, output The file NAME was deleted; otherwise, output The file NAME not found.

Inputting exit should terminate the program.

Example

Below is an output example of the described program. Try to output all the cases, like in the example.

The greater-than symbol followed by space (> ) represents the user input. Note that it's not the part of the input.

> add file1
The file file1 added successfully
> add file1
Cannot add the file file1
> add file11
Cannot add the file file11
> get file1
The file file1 was sent
> get file2
The file file2 not found
> get abcd1
The file abcd1 not found
> delete file2
The file file2 not found
> delete file1
The file file1 was deleted
> exit
